# Database Schema

## Room Version: 8

**Prepopulated DB**: `assets/databases/openbible.db` (~137MB)
- Generated by `data/import_bible.py`
- Contains 218K+ verses across 7 translations (KJV, WEB, ASV, YLT, BBE, DRA, Geneva)
- NKJV: 0 verses in prepopulated DB (copyrighted, shipped separately as `assets/translations/nkjv.db`)
- Gitignored — must be regenerated from pipeline

## Entity Inventory (20 entities)

### Core Bible
| Entity | Table | Notes |
|--------|-------|-------|
| `TranslationEntity` | translations | 7 translations: kjv, web, asv, ylt, bbe, dra, geneva, nkjv |
| `BookEntity` | books | Per-translation book metadata |
| `VerseEntity` | verses | 218K rows, all translations in one table |

| `verses_fts_bbe` | BBE full-text index |

## CRITICAL: Prepopulated DB Version
The `createFromAsset()` path copies `openbible.db` from assets. Room checks `PRAGMA user_version` against the expected version (8). If the asset DB doesn't have all entity tables (FTS5, location_events, parallel_traditions), Room throws `IllegalStateException` on first open.

**The data pipeline (`data/import_bible.py`) now generates the DB at v8** with all tables. Anyone regenerating the DB must re-run the pipeline.
